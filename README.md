## Overview

SynthTrade is a complete, end-to-end algorithmic trading intelligence system built from the ground up in Python. It is not a wrapper around an existing trading bot framework. Every component — the data ingestion layer, the feature engineering pipeline, the neural network architecture, the backtesting engine, and the live dashboard — was designed and built specifically for this system.

The core idea is simple: most retail traders lose money not because they lack a strategy, but because they cannot execute that strategy consistently, at scale, across multiple assets, without emotional interference. SynthTrade automates the analytical layer. It watches 10 assets simultaneously, 24 hours a day, runs a trained neural network on every new candle, and only surfaces a signal when the model's confidence clears a 72% threshold. Everything below that is suppressed as noise.

The system was designed with a specific trading philosophy in mind: **fewer, higher-quality signals, with strict risk-reward accounting**. A signal is only labelled as valid during training if price moves 1.5x ATR in the predicted direction without first hitting 1.0x ATR against it. This means the model is trained from the start to only learn patterns that offer a genuine edge.

## Features

### Data Ingestion
- **Deriv WebSocket API client** for live and historical synthetic index data (VIX75, Crash/Boom, Step Index)
- **yfinance integration** for real market data (US100, US30, USDJPY, GBPJPY, EURJPY, XAUUSD, XAGUSD)
- Parquet-based disk caching with configurable staleness thresholds
- Unified DataFrame format so the feature pipeline sees identical structure regardless of data source
- Async WebSocket client with auto-reconnection, tick buffering, and graceful shutdown

### Feature Engineering
- **Heikin Ashi transformation** with 11 derived features including body size, wick ratios, consecutive candle counts, and doji detection
- **Full indicator stack** computed on HA candles: EMA 9/21/50 with slope and alignment scoring, Parabolic SAR with flip detection, ATR-14, RSI-14 with momentum, Bollinger Bands with regime classification, and volume ratio with spike detection
- **40 engineered features** total, all scale-invariant through ATR normalisation so a single model generalises across assets with vastly different price levels
- **Composite Trend Score** (-3 to +3) providing instant confluence measurement across all indicators

### Machine Learning
- **Hybrid CNN-LSTM architecture** (218,819 parameters): Conv1D layers extract local candlestick patterns and indicator crossovers; LSTM layers capture trend momentum and sequential context
- **Walk-forward validation** with 5 expanding folds ensuring every evaluation uses only out-of-sample data
- **SMOTE oversampling** to balance the severe class imbalance (82% No Trade, 10% Long, 8% Short) without introducing synthetic noise into validation sets
- **Class-weighted loss function** that penalises missed genuine signals 3x harder than missed No Trade predictions
- **Confidence threshold filtering**: only signals above 72% model confidence are surfaced to the dashboard

### Backtesting Engine
- **Vectorised backtester** with ATR-based dynamic SL/TP, fixed fractional position sizing (1% equity risk per trade), and spread/commission simulation
- **Full performance metrics**: Win rate, Profit Factor, Expectancy (R), Sharpe Ratio, Sortino Ratio, CAGR, Calmar Ratio, Max Drawdown, Max Consecutive Losses
- **Walk-forward out-of-sample backtesting**: every reported trade was generated by a model that had never seen that data during training
- **Regime analysis**: performance breakdown by trending vs. ranging market conditions

### Self-Improving Architecture
- **IncrementalDataStore**: logs every real trade outcome (TP hit, SL hit, expired) as a new labeled data point
- **IncrementalRetrainer**: when enough outcomes accumulate, retrains the model on merged historical + new data and only deploys the new model if it outperforms the current one on validation loss
- **Win rate monitoring**: if an asset's win rate drops below threshold over the last 20 signals, it is automatically flagged as low-confidence and signal sizing is reduced
- The system gets measurably better the more it trades

### Live Dashboard
- **Dark trading terminal aesthetic** with IBM Plex Mono and Syne typography
- **Live Signal Feed** with per-signal confidence bars, entry/SL/TP prices, and ATR-based R:R display
- **Equity Curve** with drawdown highlighting and live annotation
- **Asset Heatmap** showing trend scores across all active assets
- **Confidence Monitor** with stacked probability area chart and threshold line
- **PnL Distribution** histogram
- **Signal Log** table with colour-coded direction
- Configurable confidence threshold and risk per trade via sidebar sliders

### Signal Flow

```
Raw OHLCV (Deriv WS / yfinance)
        ↓
Heikin Ashi Transformation
        ↓
Indicator Stack (EMA, SAR, ATR, RSI, BB, Volume)
        ↓
Feature Normalisation (scale-invariant, ATR-relative)
        ↓
Sliding Window Sequences (30 candles × 40 features)
        ↓
CNN-LSTM Model (3-class: SHORT / NO_TRADE / LONG)
        ↓
Confidence Filter (threshold: 72%)
        ↓
Signal Suppressed         Signal Emitted
(NO_TRADE)                Entry + SL + TP (ATR-based)
                                ↓
                        Dashboard + Trade Log
                                ↓
                        Outcome Observed (TP/SL)
                                ↓
                        IncrementalDataStore
                                ↓
                        Model Retrain (if threshold met)
```

## Usage

All operations are run from the project root via `main.py`.

### Launch the dashboard (demo mode)
```bash
streamlit run synthtrade/dashboard/app.py
```
Opens at `http://localhost:8501`. In demo mode the dashboard displays generated signals and demo data. All panels are fully functional.

### Train a model for a specific asset
```bash
python main.py --mode train --asset US100
```
Fetches historical data, runs the full feature pipeline, trains the CNN-LSTM with walk-forward validation, and saves the best model to `synthtrade/models/saved/`.

### Train all active assets
```bash
python main.py --mode train --asset ALL
```

### Backtest a trained model
```bash
python main.py --mode backtest --asset US100
```
Generates a full performance report in `synthtrade/backtest/results/` including trade log CSV and equity curve.

### Generate a live signal (single run)
```bash
python main.py --mode signal --asset XAUUSD
```
Fetches the latest candles, runs the model, and prints the current signal with confidence score.

## Model Details

### Architecture
```
Input: (30 timesteps × 40 features)
    ↓
Conv1D(64, kernel=3) + BatchNorm + ReLU
Conv1D(128, kernel=3) + BatchNorm + ReLU
MaxPooling1D(2) + Dropout(0.3)
    ↓
LSTM(128, return_sequences=True) + Dropout(0.3)
LSTM(64) + Dropout(0.24)
    ↓
Dense(64) + BatchNorm + ReLU + Dropout(0.15)
    ↓
Dense(3, Softmax)  →  [P(SHORT), P(NO_TRADE), P(LONG)]
```

### Training Configuration
- **Optimiser**: Adam with gradient clipping (clipnorm=1.0)
- **Loss**: Sparse categorical crossentropy with class weights
- **Early stopping**: patience=15 on val_loss, restores best weights
- **LR schedule**: ReduceLROnPlateau (factor=0.5, patience=7, min_lr=1e-6)
- **Batch size**: 32
- **Max epochs**: 80 (typically stops at 30-50 via early stopping)

### Feature Engineering
All 40 features are computed on Heikin Ashi candles rather than raw OHLCV. This keeps the smoothing effect of HA consistent across both price and indicator inputs, which is especially beneficial for noisy synthetic indices like VIX75.

Price-derived features (EMA values, BB bands, SAR, HA OHLC) are normalised as ratios to the current HA close, making the model scale-invariant. A single trained model can process assets with prices ranging from 1.5 (USDJPY at 150) to 20,000 (US100) without retraining.

## Self-Improvement

SynthTrade is designed to improve over time through a continuous learning loop:

1. **Signal generated**: The live model emits a signal with a confidence score.
2. **Trade observed**: Once price reaches TP, SL, or the max hold candles expire, the outcome is recorded.
3. **Outcome logged**: `IncrementalDataStore` stores the candle features and outcome as a new labeled data point.
4. **Retrain trigger**: After 200 new outcomes accumulate, `IncrementalRetrainer` merges new data with the original training set and retrains.
5. **Gated deployment**: The new model is evaluated against the current model on a held-out validation set. It only replaces the live model if it improves validation loss. Otherwise the current model is kept.
6. **Performance tracking**: Every retrain cycle is logged with the old and new validation metrics, providing a full audit trail of the model's improvement over time.

This means the system adapts to regime changes (trending vs. ranging markets, volatility shifts) without requiring manual retraining.

## Configuration

All system parameters are centralised in `config.py`. Key parameters:

```python
# Signal quality
SIGNAL_CONFIDENCE_THRESHOLD = 0.72   # Minimum confidence to emit a signal
MIN_REWARD_ATR  = 1.5                # Minimum target move in ATR multiples
MAX_RISK_ATR    = 1.0                # Maximum adverse move before labelling 0

# Risk management
SL_ATR_MULTIPLE = 1.0                # SL distance in ATR multiples
TP_ATR_MULTIPLE = 1.5                # TP distance in ATR multiples

# Model
SEQUENCE_LENGTH = 30                 # Lookback window (candles per input sequence)
EMA_PERIODS     = [9, 21, 50]        # EMA horizon stack
```
